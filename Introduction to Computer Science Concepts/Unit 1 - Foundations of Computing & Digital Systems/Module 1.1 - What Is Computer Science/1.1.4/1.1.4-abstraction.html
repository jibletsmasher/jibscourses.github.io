1.1.4 — Abstraction

Students explore how to manage complexity by hiding unnecessary details, interact with layered models to see what information matters for different tasks, and infer hidden rules by testing input-output patterns.

Set the Lesson Goal

Identify the main goal of abstraction.

Quick goal

Learn how to manage complexity by using abstraction to hide details that don't matter right now.

See the full lesson goals
  1. Identify which details matter for a given model or purpose.
  2. Compare detailed and simplified representations of the same system using abstraction.
  3. Infer a hidden rule from examples and test the hypothesis.
  4. Use computing terms such as layer, black box, and inference to justify claims.
