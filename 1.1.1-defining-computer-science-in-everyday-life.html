1.1.1 — Defining Computer Science in Everyday Life

Set the Lesson Goal

Predict what counts as computer science before you inspect the scenes.

Objectives

  1. Identify examples of computing embedded in ordinary environments.
  2. Distinguish computational systems from non-computational tools using evidence.
  3. Translate an everyday process into an ordered set of actionable steps.
  4. Use computing terms such as input, data, rules, output, and sensor to justify claims.

Warm-up prompt

In this lesson, a system input, a rule, and an output matter more than whether you can see code running on a screen.

    Human Algorithm

    Plan the route, test it, then explain the dependency that mattered most.

    What is going on?

    A helper bot starts at the classroom door and needs to reach the laptop in the lab area. The bot only follows the commands you give it, in order, and it stops as soon as a step is missing or out of place.

    Goal: Build a sequence that gets the helper bot from the classroom door to the laptop.

    First think through the route, then build it, run it, and revise it based on the feedback.

    Plan first

    Before you run the bot, answer these two questions so you have to think about the route dependencies instead of guessing.

    1. Before the bot can turn toward the lab wing, what must already be true?
    2. If the bot reaches the lab door and then repeats the step 'Move out to the hallway,' what should you expect?
    Answer both planning questions to unlock Run.

    Command bank

    Select the commands and arrange them in the correct order, then run the bot. You can run the bot even if some later slots are blank.

    Build the route

    Place commands in order. The route board updates immediately when you add, move, or remove a command.

    You can arrange cards now, but Run unlocks only after the planning check passes.

    Live route preview

    The board shows where the bot would be after the commands currently in the lane.

    Goal: Build a sequence that gets the helper bot from the classroom door to the laptop.

    Answer the planning questions first, then run the bot to test your sequence.

    Reflect

    Which instruction or dependency was easiest to overlook, and how did the run feedback help you revise the sequence?

    Run a working sequence, then save a short explanation to complete Step 3.

    Wrap Up

    Use the evidence from both activities and transfer the idea to a new system.

    Exit check

    1. Which item is the strongest evidence that a system is computing?
    2. Why was the plain lamp in the home scene not the best example of computer science?
    3. What made the hallway bot sequence work?
    4. A smart refrigerator shows what food is inside, reminds you when milk is close to expiring, and sends a phone alert if the door stays open. Why is that a strong example of computing?

    Reflection

    Choose one computing system from this lesson or from your own life. Explain why it counts as computing by naming at least two of these ideas: input, data, rules, output, or sensor.

    Write at least 80 characters and use at least 2 concept terms from this lesson: input, data, rules, output, sensor.

    Answer all four questions and complete the reflection, then submit the wrap-up.
